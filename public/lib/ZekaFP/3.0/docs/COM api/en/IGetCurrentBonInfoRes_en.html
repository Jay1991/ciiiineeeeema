interface IGetCurrentBonInfoRes

Remarks:

The IGetCurrentBonInfoRes interface is the result of execution of the following IZekaFP methods: GetCurrentBonInfo()

Properties:

flags - check this property. If the value 0 (zero) all other data is invalid. The most common reason is there is no open fiscal receipt.

purchases - total number of purchases in the receipt

taxgrp1 - sum for tax group 'A'

taxgrp2 - sum for tax group 'B'

taxgrp3 - sum for tax group 'C'

taxgrp4 - sum / percents per tax group 'D' (for Romania only)

taxgrp5 - sum / percents per tax group 'E' (for Romania only)

novoid - true if void is forbidden, otherwise false.

dds - true if the VAT is printed, otherwise false

detailed - true if details are printed,otherwise false

paystarted - true if there is a payment started, otherwise false

paid - true if the payment is finished, otherwise false

Remarks:

The values of the taxgrp4 and taxgrp5 properties are filled only then you work with Romanian fiscal printer. To choose the Romanian fiscal printer type you need to set the property zfpType = 1 of IZekaFP interface (Bulgarian fiscal printer is setup by default)
